AdeyWorld · Engineering Spec

Game engines, designed for
backend & app.

Six interchangeable engines, one shared contract. The backend validates & serves content and owns the kid-economy; the app plays it and reports a result. Same JSON, so a React or Flutter client uses it unchanged.

6 engines 3 difficulties · Easy always free Progress never goes backward REST · framework-neutral

How a round flows

1 Fetch

App asks for an activity at a difficulty.

GET /activities/:id?diff=easy
2 Play

Engine renders the content + difficulty config, child plays.

render(EnginePlayProps)
3 Report

Local stars for instant reward, then POST the result.

POST /play/:id/result
4 Reward

Server applies never-backward rules, returns deltas.

{ award, newBest, streak }
2

The three rules everything obeys

Never backward

Best score per (activity, difficulty) only ever increases. Stars only go up. Badges, once earned, are never lost.

Easy is free

The Easy level is always playable — the sampler that sells Premium. Medium & Hard need an unlock (rewarded video or Premium).

No fail state

Every finished game awards at least one star. Difficulty changes the challenge and the reward multiplier (×1 / ×2 / ×3), never punishes.
